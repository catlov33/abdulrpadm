# Публикация сайта «Расписание пингов глобалов» на GitHub Pages

Как выложить статический сайт из папки с редактором так, чтобы он открывался по адресу вида `https://ВАШ_ЛОГИН.github.io/ИМЯ_РЕПО/...`.

## Что лежит в репозитории

В каталоге **`docs/global-reminders/`** — страница, стили и скрипт редактора; рядом можно положить **пример** выгрузки ролей для теста.

Актуальный список ролей с сервера выдаёт **бот для первого сервера** командой **`/global_reminders_dump_roles`**: сохрани вложение **без переименования** в ту же папку, что и страница, закоммить и запушь.

В корне **`docs/`** есть служебный файл с именем на **`.nojekyll`** (точка в начале) — он отключает Jekyll при публикации **из ветки**.

### Не получается создать или запушить файл `.nojekyll` («file is hidden»)

На Windows такие имена считаются скрытыми.

- **GitHub в браузере:** *Add file* → *Create new file* → в имени укажи путь к файлу в `docs/` с точкой в начале имени → пустое содержимое → Commit.
- **PowerShell:** создай файл командой `New-Item`, затем `git add -f` для этой записи.
- Если публикуешь **только** через **GitHub Actions** (статическая выкладка без Jekyll), этот служебный файл **не обязателен** — он нужен для режима *Deploy from a branch* → папка `/docs`.

## Ошибка CI: Jekyll / «нет каталога docs»

1. **В удалённом репозитории нет папки `docs`.**  
   Скопируй из проекта всю папку **`docs/`** целиком, закоммить и запушь.

2. **Включён лишний workflow на Jekyll.**  
   В **Settings → Pages** выбери **GitHub Actions** и используй workflow из этого репозитория (статическая выкладка папки с редактором). Старый workflow со сборкой Jekyll **отключи или удали**.  
   Либо: публикация **из ветки** → папка **`/docs`** + служебный файл **`.nojekyll`** в `docs/`, чтобы не запускался Jekyll.

## Требования

- Аккаунт на [GitHub](https://github.com)
- Репозиторий с этим кодом или только с папкой сайта

## Вариант A: сайт внутри `docs/`

1. Репозиторий → **Settings** → **Pages**.
2. **Source**: *Deploy from a branch* → ветка (например `main`) → папка **`/docs`**.
3. Корень сайта = содержимое **`docs/`**. Главная страница — **`index.html` в `docs/`** (в проекте он перенаправляет в подпапку с редактором).

Адреса:

- корень репозитория на Pages → редирект на редактор;
- прямой путь к редактору — подпапка **`global-reminders/`** в URL.

Если на корне **404** — убедись, что в `docs/` есть **`index.html`**, или открывай URL с **`/global-reminders/`**.

**Важно:** на GitHub нужна вложенность **папка `docs` → папка с редактором → файлы**, а не одно длинное имя папки с косой чертой в названии.

## Вариант B: отдельный репозиторий только под сайт

Скопируй **содержимое** папки с редактором **в корень** нового репозитория (чтобы главная страница лежала в корне). В Pages: ветка `main`, папка **`/ (root)`**.

## Проверка локально

Открытие страницы через **file://** часто ломает загрузку списка ролей. Проверяй на опубликованном URL или локальным сервером, например:

```bash
python -m http.server 8080
```

## Связка с ботом

1. В редакторе собери расписание → **Скачать файл**.
2. В Discord: **`/global_reminders_import`** — прикрепи скачанный файл.
3. Список в боте: **`/global_reminder_list`**.

Часовой пояс по умолчанию задаётся в настройках **бота для первого сервера** (конфиг сервера).

## Частые проблемы

| Проблема | Что сделать |
|----------|-------------|
| Ошибка Jekyll / нет папки `docs` | Запушь **`docs/`**; отключи лишний workflow или используй ветку + `.nojekyll` |
| 404 на Pages | Подожди минуту; проверь ветку и папку источника |
| Роли не грузятся | Положи **выгрузку от бота** рядом со страницей, имя как во вложении |
| Импорт в бота не срабатывает | Небольшой файл-вложение; внутри должен быть **массив правил** в ожидаемом формате экспорта |
